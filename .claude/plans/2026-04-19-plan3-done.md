     pnpm --filter @aide/web e2e

   Playwright will auto-boot api+web via its `webServer` config. First run

   ```sh
   git checkout main
   git merge feat/ui-docker-e2e
   git tag v0.2.0
   git push origin main v0.2.0
   ```
   The `release.yml` workflow will publish
   `ghcr.io/hanfour/aide-{api,web}:v0.2.0 + :latest` and draft the GitHub
   release. The tag push is intentionally left for the operator — don't tag
   automatically.

## Known follow-ups

- Workspace `dist/src/*.js` pattern is a consequence of tsconfig including
  both `src/**` and `tests/**`. Consider a per-package `tsconfig.build.json`
  that sets `rootDir: ./src` so published packages use `dist/index.js` —
  would let us drop the post-deploy package.json rewrite in Dockerfile.api.
- E2E parallel workers requires per-worker DB isolation (the plan's known
  loose edge). Single-worker is fine for a 5-spec v0.2.0.
- `/docs` Swagger still deferred.
